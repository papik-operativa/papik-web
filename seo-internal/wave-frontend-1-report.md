## 2. Block components adopted

Block components introduced into the generator output:

- `.block-hero` (full-bleed image + overlay + content stack) — homepage / service / landing / community
- `.block-hero__media`, `.block-hero__overlay`, `.block-hero__content`,
  `.block-hero__eyebrow`, `.block-hero__title`, `.block-hero__sub`,
  `.block-hero__subtitle`, `.block-hero__cta`, `.block-hero__breadcrumb`
- `.block-hero--dark`, `.block-hero--service`, `.block-hero--home` modifiers
- `.article-hero` + `.article-hero__cat` + `.article-hero__title`
- `.article-cover` + `.article-cover__img`
- `.block-cta` + `.block-cta--closing` (closing CTA strip on every non-legal page)
- `.block-cta__label`, `.block-cta__title`

All other block-* classes from the original design system (block-info-repeatable,
block-services, block-categories, block-columns, block-info, related-card,
fitxa, despesa, sec__grid, etc.) are now styled by `blocks.css` and ready to
be emitted by future renderer extensions when richer page-type templates are
written (Wave Frontend-2 work).

## 6. Visual changes vs previous output

**Hero (homepage / service / landing / community):**
- Before: `<section class="hero hero--dark">` with a flat green gradient
  background and centred text.
- After: `<section class="block-hero block-hero--dark block-hero--home">` with
  a full-bleed photo (`/img/fotos/photo-N.jpg` rotation, `/img/og/og-*.jpg`
  for services) behind a `.block-hero__overlay` and a centred content stack.
  CTAs use the original button system (`btn-white` / `btn-outline`) sized to
  the block-hero scale.

**Hero (article / legal):**
- Before: dark hero with white-on-green text.
- After: light editorial `.article-hero` with breadcrumb, category badge
  (`.article-hero__cat`), large H1, immediately followed by an
  `.article-cover` image strip — matches the legacy
  `article-pressupost-casa-passiva.html` markup pattern.

**Closing CTA:**
- New: every non-legal page now ends with a green `.block-cta--closing`
  strip ("Tens alguna pregunta? · Parla amb nosaltres sobre el teu projecte ·
  Contactar →"), localised CA/ES/EN. This mirrors the pattern that appears
  at the foot of every page in the legacy site (`projecte-*.html`,
  `article-*.html`, `index.html`).

**Image references:**
- Homepage hero → `/img/fotos/photo-1.jpg`
- Service heroes → `/img/og/og-{construccio|rehabilitacio|promocio|patrimonis|comunitats}.jpg`
  with fallback `/img/fotos/photo-2.jpg`
- Article covers → `/img/og/og-{slug}.jpg`
- Landing heroes → `/img/fotos/photo-{1..6}.jpg` (deterministic hash on slug for variety)

A graceful `.ph .ph-hero` gradient placeholder sits beneath each
`<img>` so the design degrades cleanly when an asset is missing — no broken
icons, just the dark gradient (matching the original design's behaviour).

## 8. Pending work for Wave Frontend-2

The hero refactor is the highest-impact visual change. Several richer
block-* patterns from the original design are styled by `blocks.css` but
NOT yet emitted by the generator. They require deeper renderer
specialisation per page type:

1. **Homepage body sections** — adopt
   `.block-info-repeatable` (alternating image+text rows), `.block-services`
   (services divisions list with hover-swap GIF), `.block-categories` (3-up
   service tiles), `.block-columns` (3-up project gallery), `.block-info`
   (dark-green CTA panel) instead of the generic
   `.section / .card-grid` markup currently emitted from BLOC fences.
2. **Service pages** — wrap each H2 BLOC in `.block-info-repeatable__item`
   with alternating media columns.
3. **Article body** — promote any `[Frase de tancament]` markers to
   `.big-text` pull-quote markup with the accent border (already
   styled but not yet emitted).
4. **Project pages** — currently authored by hand in `/public/projecte-*.html`
   and not regenerated. Decide whether to migrate them into the
   `seo-internal/copy/` tree under a new `projects/` subdir so they pick up
   the `block-hero` + `fitxa` + `despesa` markup the generator now supports
   via `blocks.css`.
5. **Hero CTA hrefs** — currently hard-coded to `#contacte` and `#main`.
   Wave Frontend-2 should wire them to the localised
   `pressupost`/`presupuesto`/`budget` slug for the primary CTA on
   service / landing heroes.
6. **Real photography** — F1-images agent is migrating the project gallery;
   homepage and service heroes will need curated photography (currently
   pointing at `/img/fotos/photo-1.jpg` etc., which may not yet exist for
   every variant). Until then the `.ph .ph-hero` gradient placeholder fills
   the gap.
7. **Closing CTA hrefs** — currently `#contacte`. Wave Frontend-2 should
   point them to the localised contact slug
   (`#contacte` / `#contacto` / `#contact`) or a dedicated contact
   anchor on each page.

## 9. Idempotency check

Re-running `python3 generate_html.py` produces byte-identical output (the
generator is fully deterministic — slug hashing for landing photo rotation
uses `sum(ord(c))` on the slug string; no timestamp or random seed). Safe
to re-run as part of the deployment runbook without breaking the build.
